1. System Prerequisites Scanner CLI Tools

PRAS requires that both Trivy (for vulnerability scanning) and Semgrep (for static pattern verification) are installed globally on your machine.

1. Install Trivy

Run this command in PowerShell to automatically install Trivy via winget:

PowerShell 
winget install aquasecurity.trivy

* Note: If winget is not available, download the binary from the github releases tab, unzip it, and add the folder path to your PATH environment variable.

2. Install Semgrep

PowerShell 
pip install semgrep

1. Install Trivy & Semgrep via Homebrew

Terminal (Zsh) 
brew install aquasecurity/trivy/trivy
brew install semgrep

1. Install Trivy (Debian/Ubuntu)

Bash 
sudo apt-get install wget apt-transport-https gnupg lsb-release
wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | sudo apt-key add -
echo "deb https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main" | sudo tee -a /etc/apt/sources.list.d/trivy.list
sudo apt-get update
sudo apt-get install trivy

2. Install Semgrep

Bash 
pip install semgrep
ℹ️
After installing both tools, verify your installation paths are correct by executing trivy --version and semgrep --version in a fresh terminal window.

2. Virtual Environment & Packages Virtualenv

Isolate your python dependencies. Initialize a virtual environment in the root workspace folder, then fetch required package locks.

1. Create and Activate Virtual Environment

PowerShell 
python -m venv venv
.\venv\Scripts\Activate.ps1

1. Create and Activate Virtual Environment

Bash 
python3 -m venv venv
source venv/bin/activate

2. Install Dependencies

Install core analysis components along with baseline package versions:

Terminal 
# Install scanning engine core
pip install -r requirements-core.txt

# Install baseline vulnerable package versions
pip install -r requirements.txt

# (Optional) Install Neo4j driver requirements
pip install -r requirements-graph.txt

3. Environment Configs .env Configuration

PRAS avoids hardcoding credentials. Fill out the fields below to dynamically generate a .env file configuration.

.env 
# Outbound API configurations
GOOGLE_API_KEY=""
GITHUB_TOKEN=""

# Knowledge Graph settings
NEO4J_URI="bolt://localhost:7687"
NEO4J_USER="neo4j"
NEO4J_PASSWORD="demo-password"

Save this code inside a file named .env in the root folder of the project.

4. Run Scanner & Command Builder Interactive Command Builder

Configure your target parameters to dynamically construct the exact execution command line.

Offline Mode Use offline cached files (no APIs)

5. Optional Graph Setup Docker / Neo4j

To visually view call chains (exposed routes mapping to vulnerable APIs), you can run a local Neo4j database container.

1. Start Neo4j Database Container

Terminal 
# Start Neo4j container in the background
docker compose up -d

2. Access Database Panel

Once the container starts, open the Neo4j UI panel in your browser:

Browser Link 
http://localhost:7474

Log in using username neo4j and password demo-password.

6. Troubleshooting & FAQ Common Issues

Trivy command fails with: 'trivy not found' error ▼
This means the Trivy executable path is not added to your system environment variables. Make sure you install Trivy via winget or download the release zip, unpack it, and add the folder path containing trivy.exe to your system PATH variable, then restart your terminal.
GitHub API calls fail with Rate Limit errors ▼
GitHub restricts anonymous IP requests to 60 calls/hour. If you run multiple live scans, you will hit this limit. Fix this by adding a personal access token under GITHUB_TOKEN inside your .env file.
Do I need to run a Neo4j database to use the tool? ▼
No, Neo4j is completely optional. If the Neo4j database is not running or fails to connect, the pipeline degrades gracefully and runs all reachability queries locally using the JSON snapshot (graph_snapshot.json).
