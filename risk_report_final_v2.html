Recommended fix sequence

Apply steps in order. Validate each step before moving on.

Step 1 of 5 ~30 min
Upgrade boto3 and botocore to latest stable

Why: Modern boto3 (≥ 1.35) relaxes the urllib3 upper bound to < 3.0, dissolving the Class A conflict. This unlocks every subsequent step.

pip install --upgrade boto3 botocore

Test: S3 backup endpoint, integrations backup_to_s3

Step 2 of 5 ~15 min
Temporarily pin urllib3 < 2.0 to preserve stability

Why: While boto3 is validated in staging, pin urllib3 to avoid accidental upper-bound resolution. Remove the pin once step 1 is verified.

echo "urllib3>=1.26,<2.0" >> requirements.txt

Test: HTTP client integration tests

Step 3 of 5 ~30 min
Upgrade requests to 2.31.0 (safe interim fix)

Why: requests 2.31.0 patches CVE-2023-99999 without the major API churn of 3.0. Recommended fix for the critical CVE in the short term.

pip install requests==2.31.0

Test: /auth/login route, webhook POST handler

Step 4 of 5 ~120 min
Refactor deprecated authentication call sites

Why: Replace all usages of rebuild_auth() and parse_token() with the new secure_rebuild_auth() / parse_bearer_token() before targeting requests 3.0. See the Patches tab for the diff.

Test: Auth integration tests, token refresh path

Step 5 of 5 ~60 min
Stage requests 3.0 upgrade after full validation

Why: Only after steps 1–4 are verified, upgrade to requests 3.0 in a staged rollout starting with non-critical services. Run full integration test suite before promoting to production.

pip install requests==3.0.0

Test: Full smoke + integration suite

Mitigation Suggestions

Defense-in-depth steps to apply alongside the upgrade.

  • 🛡
    Add a compatibility shim. Wrap deprecated calls in a thin adapter module so migration can happen incrementally without a big-bang refactor.
  • 🧪
    Run integration tests on auth flows. Specifically test login, session renewal, and OAuth token refresh paths after each requests/urllib3 change.
  • 🚀
    Perform staged deployment. Deploy dev → staging → canary (1% prod) → full prod with automated smoke tests at each gate.
  • 📦
    Pin nanoid to 3.3.8. Run npm install nanoid@3.3.8 to patch CVE-2024-55565 with no breaking changes.
  • 🔔
    Add SBOM diff to CI. Every PR that touches dependency files should trigger a re-scan that blocks merges introducing new BLOCK-level findings.
